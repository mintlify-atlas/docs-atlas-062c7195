---
title: "Quickstart"
description: "Build and connect the CDLI MCP Server in under 5 minutes."
---

The CDLI MCP Server connects any MCP-compatible client (such as Claude Desktop) to the [Cuneiform Digital Library Initiative](https://cdli.mpiwg-berlin.mpg.de/) public API. It uses `stdio` transport and exposes six tools for searching artifacts, retrieving metadata, and more.

## Prerequisites

- [Node.js](https://nodejs.org/) v18 or higher
- npm

## Build the server

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url>
    cd cdli-mcp-server
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Build the TypeScript source">
    ```bash
    npm run build
    ```

    This compiles the TypeScript source into `build/index.js`. You must re-run this command after every source change.

    <Note>
      `npm run dev` compiles and immediately runs the server — useful during development, but not suitable for use with MCP Inspector or Claude Desktop (see below).
    </Note>
  </Step>
</Steps>

## Connect to Claude Desktop

<Steps>
  <Step title="Locate the Claude Desktop config file">
    <CodeGroup>

    ```text macOS
    ~/Library/Application Support/Claude/claude_desktop_config.json
    ```

    ```text Windows
    %APPDATA%\Claude\claude_desktop_config.json
    ```

    ```text Linux
    ~/.config/Claude/claude_desktop_config.json
    ```

    </CodeGroup>
  </Step>

  <Step title="Add the CDLI server entry">
    Open the config file and add the `cdli` entry under `mcpServers`. Replace the path with the absolute path to `build/index.js` on your machine.

    ```json claude_desktop_config.json
    {
      "mcpServers": {
        "cdli": {
          "command": "node",
          "args": ["/absolute/path/to/cdli-mcp-server/build/index.js"]
        }
      }
    }
    ```

    <Warning>
      Use an **absolute path**. Relative paths will cause Claude Desktop to fail silently when launching the server.
    </Warning>
  </Step>

  <Step title="Restart Claude Desktop">
    Quit and reopen Claude Desktop. The CDLI tools will appear in your conversation and Claude can invoke them automatically based on your prompts.

    Try asking: *"Search CDLI for tablets from the Ur III period"* or *"Get the full metadata for artifact P315278"*.
  </Step>
</Steps>

## Test with MCP Inspector

The [MCP Inspector](https://github.com/modelcontextprotocol/inspector) provides a browser UI for listing and calling tools interactively — useful for verifying the server before connecting it to a client.

<Steps>
  <Step title="Build the server">
    ```bash
    npm run build
    ```
  </Step>

  <Step title="Launch the inspector">
    ```bash
    npx @modelcontextprotocol/inspector node build/index.js
    ```

    <Warning>
      Do **not** pass `npm run dev` as the inspector command. The `dev` script writes build output to stdout, which corrupts the JSON-RPC stream and causes `SyntaxError: Unexpected token '>'` errors.
    </Warning>

    <Tip>
      To skip the auth token prompt during local development, set `DANGEROUSLY_OMIT_AUTH=true`:

      ```bash
      DANGEROUSLY_OMIT_AUTH=true npx @modelcontextprotocol/inspector node build/index.js
      ```

      Then open `http://localhost:6274` without any token query parameter.
    </Tip>
  </Step>

  <Step title="Open the inspector URL">
    The inspector prints a URL including an auth token:

    ```
    MCP Inspector is up and running at:
    http://localhost:6274/?MCP_PROXY_AUTH_TOKEN=<your-token>
    ```

    Copy and paste the **full URL** (including the token) into your browser. On WSL or Linux, the browser will not open automatically.
  </Step>

  <Step title="Connect to the server">
    In the inspector UI:

    1. Set **Transport** to `STDIO`
    2. Set **Command** to `node`
    3. Set **Args** to `build/index.js`
    4. Click **Connect**

    You can now list all tools and call them using the form interface.
  </Step>
</Steps>

## Test with raw JSON-RPC

Because the server uses stdio transport, you can pipe JSON-RPC messages directly without any client setup.

**List all available tools:**

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' | node build/index.js
```

**Fetch a single artifact by ID:**

```bash
echo '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"get_artifact","arguments":{"id":"P315278"}}}' | node build/index.js
```

**Search artifacts by keyword:**

```bash
echo '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"search_artifacts","arguments":{"query":"Ur III tablet"}}}' | node build/index.js
```

<Note>
  The server targets the live CDLI API at `https://cdli.earth`. These commands require an active internet connection.
</Note>
