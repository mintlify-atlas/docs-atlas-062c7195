The CDLI MCP Server exposes a set of tools that map directly to the [CDLI public API](https://cdli.earth). Each tool communicates with `https://cdli.earth` and returns structured JSON responses.

**Transport:** `stdio` (standard input/output)

## Error handling

All tools return a consistent error shape on failure:

```json
{
  "content": [{ "type": "text", "text": "<error message>" }],
  "isError": true
}
```

No exceptions are thrown to the client — errors are always surfaced as `isError: true` with a human-readable message in `content[0].text`.

## Tool summary

| Tool | Parameters | API Endpoint |
|------|------------|--------------|
| `get_artifact` | `id` (required) | `GET /artifacts/{id}.json` |
| `search_artifacts` | `query` (required) | `GET /search.json` |
| `get_authors` | none | `GET /authors.json?limit=20` |
| `get_publications` | `limit` (optional) | `GET /publications.json` |
| `get_provenience` | `limit` (optional) | `GET /proveniences.json` |
| `ping` | none | _(no external call)_ |
