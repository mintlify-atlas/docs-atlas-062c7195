## What is CDLI MCP Server?

The CDLI MCP Server is a [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that connects AI agents directly to the [Cuneiform Digital Library Initiative (CDLI)](https://cdli.mpiwg-berlin.mpg.de/) — one of the world's largest digital repositories of ancient cuneiform tablets and inscriptions.

By wrapping the CDLI public API as MCP tools, any MCP-compatible client (such as Claude Desktop) can search artifacts, fetch metadata, retrieve transliterations, list authors, and explore archaeological proveniences — without any custom integration work.

<Info>
  The server uses **stdio transport**, meaning it communicates via standard input/output. This makes it compatible with Claude Desktop, the MCP Inspector, and any other MCP client that supports stdio.
</Info>

## Available tools

The server exposes six MCP tools that map directly to the CDLI public API:

| Tool | Description |
|------|-------------|
| `get_artifact` | Fetch full metadata for a CDLI artifact by ID |
| `search_artifacts` | Full-text search across the CDLI artifact catalog |
| `get_authors` | List registered CDLI authors |
| `get_publications` | List publications from the CDLI database |
| `get_provenience` | List archaeological find sites |
| `ping` | Liveness check to verify the server is running |

## How it works

The server is a Node.js process that speaks the MCP protocol over stdio. When Claude Desktop (or another MCP client) calls a tool, the server makes an HTTP request to `https://cdli.earth` and returns the result as structured JSON.

```
Claude Desktop  ──stdio──►  CDLI MCP Server  ──HTTPS──►  cdli.earth API
```

No authentication is required. The CDLI API is publicly accessible.
